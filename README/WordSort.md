# Word Sort设计方案（WordSort.md）

## 1 参考游戏

### 1.1 《Bubble Word Jam》

- 美区AppStore链接：[‎Bubble Word Jam App - App Store](https://apps.apple.com/us/app/bubble-word-jam/id6754839783?l=zh-Hans-CN)；

- 玩法介绍：玩家需要找到具有相同特点的单词，将它们整理到一起，4个单词为一组，成功分类后即可消除。

- 玩法要点：单词分类，找共性。

## 2 详细玩法设计

### 2.1 设计思路

- Word Sort的设计思路，仍然采用单词分类的核心理念，玩家的主要目的，是对单词进行分类。但下方的单词，被排成了队列，只有队列最顶端的单词，才可以被玩家拖动，放到上方的分类框中。玩家在思考单词分类的同时，也需要考虑单词的选择顺序。否则可能导致没有足够的空间，无法拖动任何单词，导致关卡失败。

### 2.2 详细规则

#### 2.2.1 关卡结构

- 游戏为关卡模式，由N个关卡够成。达成通关条件，则进入下一关；否则，判定关卡失败，需要重新挑战当前关卡。

#### 2.2.2 单词队列

- 一个关卡中，包含N个备选的单词；
- 所有单词按照随机顺序分成M列，显示在屏幕下方区域；
- 如果单词数量过多，不会将所有单词都显示在屏幕内。队列下方超出UI显示范围的单词不会显示出来。随着上方单词被拖走，下方单词逐步上移，显示出后续单词。
- 只有每个队列最上方的单词可以被拖动。（后文补充拖动规则）

#### 2.2.3 分类框

- 单词分类框，位于界面上方区域。

- 每一关初始默认有2个空白的分类框。

- 当某个分类框为空白时（里面没有任何单词），玩家可以将下方单词队列中最上方的任意单词拖动到这个分类框中。当某个分类框中已经有单词时，玩家只能将同类的单词拖动到这个分类框中。

- 每个分类框，可以装填一组同类单词，一组的数量为4个单词。装满4个同类单词时，单词被消除。这个分类框被清空。玩家可以继续拖动其他单词到这个框中。

- 每一关初始默认有1个待解锁的分类框。上面显示+号。
  
  - 未解锁的分类框，不能放入单词。
  
  - 在玩法原型版本中，玩家不需要付出任何代价，直接点击+号，即可解锁这个分类框，解锁之后，即可放入单词。
  
  - 但这个分类框并不是永久解锁，重新开始当前关卡，或者进入下一关，这个分类框又会恢复锁定状态。

#### 2.2.4 临时存储框

- 临时存储框，位于界面中部；

- 每一关初始默认有3个空白的临时存储框；

- 一个空白的临时存储框可以存放一个单词；

- 玩家可以将下方单词队列中最上方的任意单词拖动到空白的临时存储框中。

- 每一关初始默认有1个待解锁的临时存储框。上面显示+号。
  
  - 未解锁的临时存储框，不能放入单词。
  
  - 在玩法原型版本中，玩家不需要付出任何代价，直接点击+号，即可解锁这个临时存储框，解锁之后，即可放入单词。
  
  - 但这个临时存储框并不是永久解锁，重新开始当前关卡，或者进入下一关，这个临时存储框又会恢复锁定状态。

#### 2.2.5 拖动规则

- 下方单词队列中，只有处于最顶端的单词，才可以拖动到上方分类框、或中间临时存储框中。

- 当队列最顶端的单词被成功放置到上方分类框中，或者中间临时存储框中，队列下方的单词依次上升。

- 当玩家将下方队列中的单词向上方分类框拖动时：
  
  - 如果将单词拖动到空白的分类框中，则直接放入框中。
  
  - 如果将单词拖动到非空白的分类框中，则判断：
    
    - 如果拖动的单词，与分类框中已存在的单词，属于同一类，则单词放入该分类框中；
    
    - 如果拖动的单词，与分类框中已存在的单词，不属于同一类，则先扣除一颗红心，然后继续判断：
      
      - 如果中间临时存储框有空位，则放入最靠前的空白临时存储框中；
      
      - 如果中间临时存储框没有空位，则单词返回原位（下方单词队列最上端）。

- 当玩家将下方队列中的单词向中间临时存储框拖动时：
  
  - 如果中间临时存储框有空位，则放入最靠前的空白临时存储框中；
  
  - 如果中间临时存储框没有空位，则单词不能放入临时存储框，而是返回原位（下方单词队列最上端）。

- 当玩家将分类框A中的单词拖动到另一个分类框B中时：
  
  - 如果目标分类框B是空白的，则可以直接放入。
  
  - 如果目标分类框B不是空白的，则判断：
    
    - 如果从A中拖动出来的单词，与B中的单词属于同一个类型，则可以将单词放入B中。
    
    - 如果从A中拖动出来的单词，与B中的单词不属于同一个类型，则不可以将单词放入B中。玩家松手时，单词返回到A中。并且，扣除一颗红心。

- 当玩家将临时存储框中的单词，拖动到分类框中时：
  
  - 如果该分类框是空白的，则直接放入单词。
  
  - 如果该分类框不是空白的，则判断：
    
    - 如果拖动的单词，与分类框中的单词属于同一个类型，则可以将单词放入。
    
    - 如果拖动的单词，与分类框中的单词不属于同一个类型，则不可以将单词放入分类框中。玩家松手时，单词返回到临时存储框中。并且，扣除一颗红心。

- 玩家可以将临时存储框中的单词，拖动到另外一个空白的临时存储框中。

- 以下的拖动操作是不允许的：
  
  - 将分类框中的单词拖动到临时存储框中。
  
  - 将分类框、或临时存储框中的单词，拖回到下方单词队列中。

#### 2.2.6 消除规则

- 当上方某个分类框中，填满4个同类型单词时，在分类框上方显示出这4个单词所属的类型名称，然后4个单词即可被消除，然后类型名称也消失。分类框恢复空白状态。

#### 2.2.7 红心规则

- 每一关初始默认有3颗红心。

- 当玩家将下方队列中的单词向上方分类框拖动时，如果拖动的单词，与分类框中已存在的单词，不属于同一类，则扣除一颗红心。

- 每次重新开始当前关卡，或者进入下一关，则恢复到3颗红心。（3颗红心是上限）

#### 2.2.8 关卡胜利条件

- 将下方单词队列中的所有单词，都通过归类的方式消除完毕，则关卡胜利。

- 关卡胜利后，弹出胜利界面。玩家点击“下一关”按钮，则进入下一关。

#### 2.2.9 关卡失败条件

- 遇到下列任意情况，则判断关卡失败：
  
  - 3颗红心都被扣除。
  
  - 同时满足：上方分类框没有空白的框、中间临时存储框没有空位、下方单词队列最上方的单词的类型与上方任何分类框中的单词都不同。

- 如果达到关卡失败条件，则弹出失败界面。玩家点击“重新开始”按钮，则重新进入这一关。

## 3 特殊机制

### 3.1 问号单词

- 在关卡中，下方单词队列中，可能存在问号单词。问号单词，不显示文本，文本被问号代替。
- 问号单词不能出现在单词队列的最顶端。问号单词上升到队列最顶端时，去掉问号，显示出正常单词文本。

#### 4.1.3 关卡表（LevelSheet）

- LevelSheet表，用于配置每个关卡中的总单词数、下方单词队列的数量、问号单词数量。

- 关键字LevelID，关卡ID，作为Key，用于索引。

- 关键字WordCount，配置本关的总单词数量。这个数量一定是4的倍数。

- 关键字ColumnCount，配置本关的单词队列数量。
  
  - 每个单词队列中的单词数量，并不一定相等。例如，总单词数量为28，队列数量为3，28不能被3整除，那么，3个队列的单词数量可以分别为：9、10、9。

### 4.2 初始生成

- 关卡内容的生成要求如下：
  
  - 读取LevelSheet，确定本关的总单词数量WordCount，单词队列数量ColumnCount，问号单词数量QMarkCount。
  
  - 在TypeSheet表中，随机选取几个类型，并选取这个类型的单词，规则如下：
    
    - 类型选取，不能重复。即，在同一个关卡中，同一个类型的单词，只能出现1组（即4个）；
    
    - 一个类型中，随机选取WordList中包含的单词，选择4个不重复的单词；
    
    - 随机选出的所有类型、所有单词总量，等于本关的WordCount。
  
  - 将随机得到的所有单词，随机排列在队列中。
  
  - 队列中，除最顶端的单词之外，随机设定为问号单词，数量为QMarkCount。

- 要保证关卡有解。

- 在初始进入某一关时，按照上述要求生成关卡，生成后保存生成结果。每次重新开始当前关卡，都采用之前保存的结果，不再重新随机生成。

### 4.3 关卡重置

- 在游戏过程中，玩家点击左上角的重置按钮，则恢复当前关卡的初始状态，重新开始当前关卡。
