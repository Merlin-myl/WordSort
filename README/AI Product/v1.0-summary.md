## 实现内容

### 1. 项目文件
- **`index.html`** (`D:\Work\AI Project\HTML Project\WordSort\index.html`)
  - 单文件HTML游戏，包含完整CSS + JavaScript
  - 文件大小：约1500行，~42KB
  - 无外部依赖，双击即可在浏览器运行

### 2. 已完成功能

#### 选关界面
- 3列×10行网格布局，显示全部30关
- 所有关卡一次性开放，点击任意关卡开始
- 带标题和副标题

#### 游戏界面
- **顶部栏**：返回按钮、关卡名称、红心(3颗)、重置按钮
- **分类框区域**：2个默认 + 1个可解锁（点+号解锁）
- **临时存储区域**：3个默认 + 1个可解锁（点+号解锁）
- **单词队列**：M列单词队列，列顶单词高亮可拖动
- 问号单词以"?"显示，到达列顶后自动显示真实文本

#### 核心玩法
- **拖拽**：同时支持鼠标拖拽(PC)和触摸拖动(手机)
- **分类判断**：拖入分类框时自动判断类型匹配
- **错误惩罚**：类型不匹配→扣1心，优先放入临时存储
- **消除动画**：4个同类单词消除时显示类型名称+消失动画
- **胜利条件**：所有单词消除完毕→弹出胜利弹窗
- **失败条件**：红心耗尽或死锁→弹出失败弹窗
- **特殊机制**：问号单词（非列顶时隐藏文本，到列顶时显示）

#### 弹窗
- **胜利弹窗**：显示"恭喜通关"，第30关显示"全部通关"
- **失败弹窗**：显示失败原因，"重新挑战"按钮
- 两弹窗都提供"返回选关"选项

### 3. 配置数据
- 136个单词（从WordSheet.xlsx导入）
- 33种类型（从TypeSheet.xlsx导入）
- 30个关卡（从LevelSheet.xlsx导入，含难度递进）

### 4. 关卡生成
- 每次进入新关随机生成（保存结果，重试不复随机）
- 可解性检查：
  - ≤32单词：完整DFS搜索（10万节点，快速）
  - >32单词：带超时的启发式DFS（50万节点，3-5秒超时）
  - 最多200次重试
- 关卡结果缓存（Map），重试同一关不重新生成

### 5. 视觉设计
- 深色主题背景（渐变蓝色系）
- 粉色系主色调（#e94560）
- 单词瓦片带阴影和动画
- 分类框类型颜色编码
- 响应式设计：手机竖屏 + 桌面端适配

## 已知限制

1. **可解性检查**：大于32词的关卡DFS可能超时，极端情况下可能生成理论无解的关卡（概率极低）
2. **队列显示**：单词数量过多时，队列底部单词不可见（通过滚动视觉提示）
3. **性能**：60词大关卡生成可能需要3-5秒
